Data-link Layer
	-다양한 네트워크환경 구성
		-버스형:이더넷의 대표적인 구성,단일노드로 구성(동축케이블) < 충돌(Collision)발생가능성, 확장성,유지보수쉬움
		-환형:페쇄형구조, 토큰링:토큰을 가진장비만 통신함. 차례를 기다림. , 확장성이 좋지않음, 
		-성형:스위치장비가 중심에있고 나머지가 연결됨, 거대한규모에도 사용가능, 중앙장비가 고장시 다른장비의 영향
		-트리형:계층적구조, 대규모네트워크망에 적합함, 구조상 관리가쉽고 확장성이 좋음, 링크문제있을시 하위 환경에 영향
		-망형:가용성을 높음 전부 1:1로 연결되어있음, 비용이 큼, 확장성이 낮고 유지보수가 힘듬. 중소규모 네트워크에 사용

	-이더넷
		-원할때 통신가능, 유지보수가 쉬움
		-FastEthernet = 100mbps
		이더넷프레임 
			-LLC : 네트워크 로컬 장비간 논리적 연결 지원			
			        연결성확인, 관리, 제어 
			        중간장비는 제어를 최대한 안하는것이 좋음. < 전송속도 저하의 이유
			-MAC : 네트워크 매체에 접근 통제
			          매체접근제어 CSMA(충돌감지)/CD(충돌제어)
	-CSMA/CD
		-하나의 장비가 데이터를 보낼때 다른장비에게 신호를 보냄
		-동시에 전송될 시 충돌발생 > JAM 신호 발생 > 통신중지 > 재전송 
		-충돌 발생가능시 알려줄수 있는 범위 Collision Domain
		-스위치가 있으면 충돌이 거의 발생하지 않음
	-프레임종류
		-이더넷2 (주로사용됨 LLC포함x) 프레임 >> (|| 동기화신호 | 기본헤더(MAC주소(목적지,출발지주소),타입) | 데이터 | 푸터(FCS:오류검출) ||)
		-IEEE 802.3 Frame (LLC포함)
	-MAC주소(EUI-48)
		-48bit / 16진수 / ':' or '-'으로 표기
		-제조회사 식별 번호(OUI) + 장치 식별 번호(NIC) 

Network Layer
	-IP(인터넷 프로토콜)
		-32bit로 구성
		-8bit(옥텟)씩으로 자름
		-Subnet-mask(net-id와 host-id를 식별해주는값이며 1은 net 0은 host로 표시한다.)
			ex 192.168.10.32
		   	    255.255.255.0 (서브넷마스크)
			Prefix-length ( 넷아이디로 사용된 부분의 bit수를 세어 표시 )
			ex 192.168.10.0 / 24
		-클래스(첫번째 옥텟의 bit만을 사용하여 Class별 사용 IP의 범위를 나눈다)
			A Class net.host.host.host  0xxx xxxx / 0~127 128개
			B Class net.net.host.host    10xx xxxx / 128~191 64개
			C Class net.net.net.host     110x xxxx / 192~223  32개 *첫번째 옥텟숫자에따라 범위를 알수있음
			D(멀티캐스트용도,그룹통신), E는 특수한경우에만 사용(네트워크 개념X)
			-클래스를 나누면 서브넷마스크가 필요가없음 < 주소의 처리가 빠름
