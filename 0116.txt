통신타입(주소에 연관있는 통신방식)
	유니케스트(1:1통신)
		-단일목적지를 지정해서 통신(목적지가 특정장비 하나만)
		-ABC클래스를 사용(유니케스트 주소)
		-대다수가 유니케스트 사용(80%)
		-다른 호스트에 부하를 주지 않음
		-다수에게 전달할 시 비효율적이다.
		-통신의 대상자가 단 하나의 경우(ex 출석)
	브로드케스트
		-동일한 정보를 한번에 모든 호스트에게 전달
		-많은 브로드케스트는 호스트의 성능저하를 가져옴 -> 적당한게좋음
		-불특정다수와 통신
		-범위 안에있는 모든장비에 통신됨
		-255.255.255.255를 브로드캐스팅주소로 사용(E클래스의 마지막주소)
	멀티케스트
		-특정다수에게 통신하는 방식 ( 유니 + 브로드)
		-그룹통신
		-공통적인 피드백의 집단을 가짐
		-불필요한 부하를 주지않음->효율적
		-계속적인 트래픽이 필요함(멀티캐스트를 하기위해 추가적으로 필요함)
IP 서브넷팅(분할)CIDR
	1기준구하기
		네트워크를 나누고자 할대 호스트기준으로 나눌건지 네트워크 기준으로 나눌건지 정한다.
	2필요한비트수
		정해진 기준값에 따라 필요한 비트수를 구한다.
	3넷아이디구분
		잘라진 지점을 기준으로 새로운 넷아이디 범위를 구한다.
	4서브넷의 수와 IP주소의 수 파악
		나눠진 서브넷의 개수는 몇이고 각 서브넷당 몇 개의 IP가 할당되는지 확인한다.
	5각 서브넷의 범위구하기
		나눠진 서브넷 당 네트워크 범위를 구하고 네트워크 대표주소와 서브넷마스크로 표기한다.
	6가용IP범위파악
		장비에 할당할 수 있는 가용가용 IP의 범위를 파악한다.

	물리적:라우터를 보고 네트워크 수 확인
	논리적:네트워크주소를 통해서 확인
	ex) 200,1,1,0/24 ~200,1,1,255/24일때 => 총 256개 - 2 = 254개의 가용IP
		200,1,1,0/24는 대표주소(IP주소 범위의 첫번째 주소를 사용하며, 네트워크 범위 전체를 나타내는 값으로 사용)
		브로드케스트주소(해당 네트워크 안의 브로드케스트 범위를 나타내기 위해 사용하며, 범위의 마지막주소를 사용)
					Directed Broadcast : 200,1,1,255/24
					Limited(local) Broadcast : 255,255,255,255
		200,1,1,1~200,1,1,254는 가용 IP주소(네트워크 범위에서 대표주소와 브로드캐스트 주소를 제외한 나머지 주소만 호스트에 적용할 수 있다.)
	
	FLSM문제풀이
	-균등 서브넷팅 방식: FLSM (하나의기준으로 한번만 나눠서 동일한 크기로 사용하는 방식)
	200.1.1.0/24를 할당받았을때
		@호스트 기준(ip기준으로 나눌때) -> 뒤에서 부터 필요한 개수계산
		ex) 25대 pc -> 27개의 IP주소 필요 (2^(5) >= 27) => 5개의 비트가 필요함 네번째 옥텟의 앞쪽 3개의 비트는 네트워크주소로 포함됨
			네트워크 1 200.1.1.0/27~200.1.1.31/27 네트워크2 200.1.1.32/27~200.1.1.63/27 ....
			네트워크 2를 IP 할당할때 가용 : 200.1.1.33~62
					        서브넷 : 255.255.255.224
			=>8개의 네트워크로 구분가능 ---.000(2^3=8개의 서브네트워크)|00000
		ex) 10대 pc
			200.1.1.0/28 
				200.1.1.0 가용 200.1.1.1~15 서브넷마스크 255.255.255.240(11110000)
				200.1.1.16
				200.1.1.32
				...
				200.1.1.239
		ex) 2대 pc
			200.1.1.0/30
				200.1.1.0 가용 200.1.1.1~3 서브넷마스크 255.255.255.252(11111100)
				200.1.1.4 가용 200.1.1.5~7 
				...
				200.1.1.251
		@네트워크 기준(덩어리로 나눌때)
		ex) 3개의 부서(2개의 비트필요) -> 앞에서 부터 필요한 개수 계산 00(필요부분)|000000
			네트워크1 200.1.1.0 / 26 서브넷마스크 255.255.255.192
			네트워크2 200.1.1.64 / 26....
	VLSM문제풀이
	ex)200.1.1.0/24할당시 A부서 100명 B부서 30명 C부서 2명 일때
		200.1.1.64로 할당가능하므로 FLSM방식 사용불가 00|000000 < 100개를 할당못함
		큰숫자부터 IP할당
		A => 200.1.1.0/25(0~127)개 할당
		B => 200.1.1.128를 나눠서 할당 => 200.1.1.128/27 (128~159)개 할당
		C => 200.1.1.160 / 30 (160~163)사용

	ex) A부서 50명 B부서 50명 C부서 10명 D부서 2명
		A=> 200.1.1.0/26 (0~63)할당 0000 0000
		B=> 200.1.1.64/26 (64~127)할당
		C=> 200.1.1.128/28 (128~143)할당 
		D=> 200.1.1.144/30 (144~147)할당
